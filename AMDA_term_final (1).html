Code 
#Goal: Create NEW, MEANINGFUL variables from existing ones.
# Think of it like deriving new metrics from raw instrument readings.
# Each new variable should answer a business/research question.

df <- df %>%
  mutate(

    # ── FEATURE 1: Entertainment Addiction Score (EAS) ─────────────────────
    # Business Reason: A composite index that measures how strongly a person
    # exhibits addictive entertainment behaviors — binge-watching, infinite
    # scrolling, and compulsive checking. Higher = more addictive pattern.
    # Research Value: Core independent variable for predicting academic harm.
    # Components: binge_watch + stream_overrun + scroll_overrun + boredom_check
    entertainment_addiction_score = (binge_watch + stream_overrun +
                                       scroll_overrun + boredom_check) / 4,

    # ── FEATURE 2: Academic Performance Index (API) ────────────────────────
    # Business Reason: Aggregates 5 academic effectiveness items into one
    # score. struggle_focus is REVERSE-CODED (5→1, 4→2, etc.) because
    # agreement with "I struggle to focus" = POOR academic outcome.
    # This index is the KEY DEPENDENT VARIABLE of the study.
    academic_performance_index = (focus_fully + complete_on_time +
                                    mentally_engaged + acad_satisfied +
                                    (6 - struggle_focus)) / 5,

    # ── FEATURE 3: Wellbeing Score ─────────────────────────────────────────
    # Business Reason: Captures sleep quality, mental fatigue, and energy.
    # mental_fatigue and latenight_sleep and habit_interfere are reverse-coded
    # (high agreement = bad outcome). Useful to study whether entertainment
    # habits erode overall wellbeing beyond just academics.
    wellbeing_score = (sleep_consistent + (6 - latenight_sleep) +
                         (6 - mental_fatigue) + feel_energized +
                         (6 - habit_interfere)) / 5,

    # ── FEATURE 4: Gaming Intensity Score ─────────────────────────────────
    # Business Reason: Games are a distinct entertainment mode (active vs.
    # passive). By isolating gaming behavior, we can compare gamers vs.
    # non-gamers on academic outcomes. Useful for sub-group analysis.
    gaming_intensity = (game_regular + game_overrun + game_absorbed +
                          game_competitive + game_improve + game_major_hobby) / 6,

    # ── FEATURE 5: Screen Time Numeric ────────────────────────────────────
    # Business Reason: The ordered factor is great for visualization, but
    # regression and correlation need numeric values. We assign midpoints
    # to each band for quantitative analysis.
    screen_time_numeric = case_when(
      screen_time == "Less than 2 hours" ~ 1,
      screen_time == "2–4 hours"         ~ 3,
      screen_time == "4–6 hours"         ~ 5,
      screen_time == "6–8 hours"         ~ 7,
      screen_time == "More than 8 hours" ~ 9
    ),

    # ── FEATURE 6: Platform Count ─────────────────────────────────────────
    # Business Reason: Multi-platform users are more exposed to entertainment
    # stimuli. Counts how many platforms each person uses from their response.
    platform_count = str_count(platforms, ",") + 1,

    # ── FEATURE 7: Entertainment Risk Category ────────────────────────────
    # Business Reason: Stakeholder-friendly label that bins respondents into
    # "Low Risk", "Moderate Risk", or "High Risk" based on EAS.
    # This is actionable — counselors can target High Risk students.
    risk_category = case_when(
      entertainment_addiction_score <= 2.5 ~ "Low Risk",
      entertainment_addiction_score <= 3.5 ~ "Moderate Risk",
      TRUE                                 ~ "High Risk"
    ),
    risk_category = factor(risk_category,
                           levels = c("Low Risk", "Moderate Risk", "High Risk"),
                           ordered = TRUE)
  )

```{r}
    # scrolling, and compulsive checking. Higher = more addictive pattern.
    # Research Value: Core independent variable for predicting academic harm.
    # Components: binge_watch + stream_overrun + scroll_overrun + boredom_check
    entertainment_addiction_score = (binge_watch + stream_overrun +
                                       scroll_overrun + boredom_check) / 4,

    # ── FEATURE 2: Academic Performance Index (API) ────────────────────────
    # Business Reason: Aggregates 5 academic effectiveness items into one
    # score. struggle_focus is REVERSE-CODED (5→1, 4→2, etc.) because
    # agreement with "I struggle to focus" = POOR academic outcome.
    # This index is the KEY DEPENDENT VARIABLE of the study.
    academic_performance_index = (focus_fully + complete_on_time +
                                    mentally_engaged + acad_satisfied +
                                    (6 - struggle_focus)) / 5,

    # ── FEATURE 3: Wellbeing Score ─────────────────────────────────────────
    # Business Reason: Captures sleep quality, mental fatigue, and energy.
    # mental_fatigue and latenight_sleep and habit_interfere are reverse-coded
    # (high agreement = bad outcome). Useful to study whether entertainment
    # habits erode overall wellbeing beyond just academics.
    wellbeing_score = (sleep_consistent + (6 - latenight_sleep) +
                         (6 - mental_fatigue) + feel_energized +
                         (6 - habit_interfere)) / 5,

    # ── FEATURE 4: Gaming Intensity Score ─────────────────────────────────
    # Business Reason: Games are a distinct entertainment mode (active vs.
    # passive). By isolating gaming behavior, we can compare gamers vs.
    # non-gamers on academic outcomes. Useful for sub-group analysis.
    gaming_intensity = (game_regular + game_overrun + game_absorbed +
                          game_competitive + game_improve + game_major_hobby) / 6,

    # ── FEATURE 5: Screen Time Numeric ────────────────────────────────────
    # Business Reason: The ordered factor is great for visualization, but
    # regression and correlation need numeric values. We assign midpoints
    # to each band for quantitative analysis.
    screen_time_numeric = case_when(
      screen_time == "Less than 2 hours" ~ 1,
      screen_time == "2–4 hours"         ~ 3,
      screen_time == "4–6 hours"         ~ 5,
      screen_time == "6–8 hours"         ~ 7,
      screen_time == "More than 8 hours" ~ 9
    ),

    # ── FEATURE 6: Platform Count ─────────────────────────────────────────
    # Business Reason: Multi-platform users are more exposed to entertainment
    # stimuli. Counts how many platforms each person uses from their response.
    platform_count = str_count(platforms, ",") + 1,

    # ── FEATURE 7: Entertainment Risk Category ────────────────────────────
    # Business Reason: Stakeholder-friendly label that bins respondents into
    # "Low Risk", "Moderate Risk", or "High Risk" based on EAS.
    # This is actionable — counselors can target High Risk students.
    risk_category = case_when(
      entertainment_addiction_score <= 2.5 ~ "Low Risk",
      entertainment_addiction_score <= 3.5 ~ "Moderate Risk",
      TRUE                                 ~ "High Risk"
    ),
    risk_category = factor(risk_category,
                           levels = c("Low Risk", "Moderate Risk", "High Risk"),
                           ordered = TRUE)
  )

```
#### Interpretation:

The engineered features capture key behavioral patterns, revealing a relatively high Entertainment Addiction Score (mean ≈ 3.64), with a majority of respondents classified as High Risk, indicating prevalent addictive usage. Academic performance remains moderate (mean ≈ 3.12), suggesting potential adverse effects of excessive entertainment engagement. Wellbeing is comparatively lower (mean ≈ 2.79), reflecting issues related to sleep and energy levels. Gaming intensity is moderate (mean ≈ 2.98), indicating it contributes to overall behavior but is not the dominant factor. Overall, the findings suggest that higher entertainment usage is associated with average academic outcomes and reduced wellbeing.

## 5.4 ONE-WAY ANOVA — Does risk category SIGNIFICANTLY affect API?

```{r}
# WHY: We want to statistically prove (not just visually suggest) that the
# 3 risk groups have meaningfully different academic outcomes.
# ANOVA tests: "Is the mean API the same across all risk categories?"
# If p < 0.05, it is NOT the same — there's a real difference.
# =============================================================================

cat("\n🔬 ONE-WAY ANOVA:\n")
cat("H₀: Mean API is equal across Low, Moderate, and High Risk groups\n")
cat("H₁: At least one group mean is significantly different\n\n")

anova_model <- aov(academic_performance_index ~ risk_category, data = df)
print(summary(anova_model))

# If ANOVA is significant, run Tukey HSD to find WHICH pairs differ
tukey_result <- TukeyHSD(anova_model)
cat("\nTukey HSD Post-Hoc Test (which pairs are different?):\n")
print(tukey_result)

```
#### Interpretation:

The ANOVA results show a statistically significant difference in academic performance across risk categories (p = 0.005), indicating that at least one group differs in mean API. The Tukey post-hoc test reveals that both Moderate Risk and High Risk groups have significantly lower academic performance compared to the Low Risk group. However, there is no significant difference between Moderate and High Risk groups. This suggests that moving from low to higher risk levels is associated with a decline in academic performance, but differences within higher risk groups are minimal.
